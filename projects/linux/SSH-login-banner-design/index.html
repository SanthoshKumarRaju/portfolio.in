SSH Login Banner Design

• local lab
Summary

The project will guide you through the process of designing and implementing a custom SSH login banner on a Linux system. You will learn how to create a banner message that is displayed to users when they log in via SSH, and how to configure the SSH server to use this banner. This is a common practice for providing important information, legal notices, or welcome messages to users before they access the system.

Linux Fundamentals

Goal: Understand what SSH login banners are, why they are used in Linux systems, and how they help improve system visibility, user experience, and security awareness when administrators access servers through SSH.

  1. Introduction to SSH Login Banners
    An SSH login banner is a message displayed to users when they connect to a Linux server using the Secure Shell (SSH) protocol. This message can appear either before authentication or after a successful login depending on how the system is configured. Login banners are commonly used to display system information, security warnings, and welcome messages.
  2. Purpose of a Login Banner
    Login banners serve multiple purposes in system administration. They provide users with important information about the server, communicate access policies, and inform users that the system is monitored and restricted to authorized use. This helps organizations maintain security awareness and professional system management practices.
  3. Improving System Visibility
    Administrators often need quick insight into the system immediately after logging in. A well-designed login banner can display useful information such as:
    • Hostname of the server
    • Operating system version
    • System uptime
    • CPU and memory usage
    • Disk space availability
    • Network or IP information
    This allows administrators to quickly understand the system state.
  4. Security Awareness
    Many organizations configure SSH banners to display security warnings that notify users that the system is restricted to authorized personnel only. These warnings may also state that system activity is monitored and logged. This helps discourage unauthorized access and supports security compliance policies.
  5. User Experience and Professional Appearance
    A structured and well-formatted login banner improves the user experience for administrators by clearly presenting important information in a readable format. It also gives the system a professional appearance, especially in production environments.
  6. Project Objective
    In this project, the goal is to design and implement a custom SSH login banner that dynamically displays system information and security notices when users connect to a Linux system. The banner will be designed to be clean, informative, customizable, and easy to maintain.
Completion check: You should understand what SSH login banners are, why they are used in Linux systems, and how they help improve system visibility, user awareness, and security communication.

Goal: Understand the limitations of default SSH login messages in Linux systems and why administrators often need more useful system information immediately after logging into a server.

  1. Default SSH Login Behavior
    When a user connects to a Linux server using SSH, the system usually displays a very minimal login message. In many systems, this message only includes basic details such as the operating system version or a short system message. While this information may be helpful, it often does not provide enough operational insight for system administrators who need to quickly understand the system's status.
  2. Limited Visibility for Administrators
    After logging into a server, administrators often need to manually run multiple commands to understand the current condition of the system. For example, they may need to check uptime, system load, memory usage, or disk space before performing any operations. This process can slow down troubleshooting and routine maintenance.
  3. Common Information Administrators Need
    Immediately after login, administrators typically want to know important system details such as:
    • System uptime
    • CPU load and performance
    • Memory usage
    • Disk space availability
    • Number of logged-in users
    • Network and IP information
    Without a structured login banner, administrators must manually execute several commands to gather this information.
  4. Inefficiency in System Administration
    Repeating the same diagnostic commands every time a user logs into a server can become inefficient, especially when managing multiple systems. In large environments with many servers, administrators benefit from having important system information displayed automatically during login.
  5. Need for a Structured Login Banner
    To solve this problem, many administrators create custom SSH login banners that automatically display useful system information when a user logs in. These banners provide a quick overview of the server's current state, helping administrators make faster decisions and improving overall operational efficiency.
Completion check: You should understand why default SSH login messages are limited and why administrators benefit from customized login banners that provide system insights immediately after login.

Goal: Understand the importance of SSH login banners and how they improve security awareness, system visibility, and communication between administrators and users who access Linux servers.

  1. Security Awareness
    One of the primary purposes of an SSH login banner is to inform users about the security policies of the system. Many organizations display warning messages stating that the system is restricted to authorized users and that all activities may be monitored or logged. These warnings help discourage unauthorized access attempts and establish clear security expectations for users connecting to the server.
  2. Legal and Compliance Requirements
    In many enterprise environments, login banners are required for security compliance and legal protection. These banners typically inform users that the system is owned by an organization and that any unauthorized access is prohibited. If a security incident occurs, these warnings can support legal and compliance investigations by showing that users were clearly notified about system policies.
  3. System Identification
    Login banners can help users quickly identify the system they are connected to. This is particularly important in environments where administrators manage multiple servers such as development, staging, and production systems. A banner displaying the hostname, environment type, or system role helps prevent accidental changes on the wrong system.
  4. Operational Visibility
    A well-designed SSH login banner can display important system information immediately after login. This allows administrators to quickly understand the current status of the server without running additional commands. Typical information displayed may include:
    • System uptime
    • CPU load and performance
    • Memory usage
    • Disk usage and available storage
    • Number of logged-in users
    • System hostname and IP address
  5. Improved User Experience
    Structured login banners improve the overall user experience for administrators and developers who frequently access servers. Instead of manually checking system details, the most important information is presented immediately during login. This makes system monitoring more efficient and saves time during daily operations.
  6. Communication Channel for Administrators
    Login banners can also be used as a communication channel between system administrators and users. For example, banners can display important notices such as maintenance schedules, system updates, or operational alerts. This ensures that users are informed about system events as soon as they log in.
  7. Professional System Presentation
    Custom login banners provide a clean and professional presentation for servers, especially in enterprise environments. A structured banner with formatted information, ASCII headers, and organized system data gives administrators a clear and consistent view of the system during every login session.

Goal: Understand why SSH login banners are important for security policies, legal compliance, and organizational governance, and how they help protect systems by clearly informing users about access restrictions and monitoring policies.

  1. Importance of Security Warnings
    SSH login banners are commonly used to display security warnings before or after a user logs into a system. These warnings inform users that the system is private property and that access is restricted to authorized individuals only. The message also communicates that activities performed on the system may be monitored, recorded, and audited.
  2. Legal Protection for Organizations
    Many organizations require login banners to establish legal protection and clarify ownership of the system. By displaying a warning message before authentication, the organization can clearly inform users that unauthorized access is prohibited. If a user proceeds with login after seeing the warning, it indicates that they have acknowledged the terms of use.
  3. Compliance Requirements
    In regulated industries such as finance, healthcare, and government sectors, login banners are often required as part of security compliance frameworks. These frameworks require organizations to notify users that the system is monitored and that improper use may lead to disciplinary or legal action. Implementing login banners helps organizations meet these compliance standards.
  4. Monitoring and Audit Awareness
    Login banners often inform users that system activities may be logged and monitored. This helps create transparency and encourages responsible use of the system. Knowing that actions are being monitored can discourage misuse and unauthorized behavior within the system.
  5. Restricting Unauthorized Access
    Security banners clearly state that only authorized users are permitted to access the system. This helps discourage attackers or unauthorized users by explicitly informing them that the system is protected and monitored.
  6. Best Practices for Security Banners
    When designing security banners, administrators should follow several best practices:
    • Clearly state that the system is restricted to authorized users.
    • Inform users that activity may be monitored or logged.
    • Avoid revealing sensitive system information.
    • Use clear and professional language.
    • Ensure the banner complies with organizational policies.
  7. Placement of Security Banners
    Security banners are typically displayed in two locations depending on the system configuration:
    • Pre-login banner: Displayed before authentication using files such as /etc/issue.net.
    • Post-login banner: Displayed after authentication using files such as /etc/motd.
    Pre-login banners are usually preferred for legal warnings because they appear before the user provides authentication credentials.

Goal: Understand the two primary types of SSH login banners—pre-login banners and post-login banners—and learn when each type should be used and how they serve different purposes in system administration.

  1. Overview of SSH Login Banner Types
    SSH login banners are generally divided into two categories based on when they appear during the login process. These are:
    • Pre-login banners
    • Post-login banners
    Each type serves a different purpose and is configured using different files and system settings within the Linux environment.
  2. Pre-Login Banner
    A pre-login banner is displayed before a user successfully authenticates to the system. This message appears immediately after an SSH connection is established but before the user enters their password or authentication credentials.
  3. Purpose of Pre-Login Banners
    Pre-login banners are primarily used for security and legal notifications. They inform users that the system is private, monitored, and restricted to authorized individuals. This ensures that users are aware of system policies before they proceed with authentication.
  4. Common Pre-Login Banner Configuration
    In most Linux systems, pre-login banners are configured using the file:
    • /etc/issue.net
    The SSH server configuration file is then updated to reference this banner file using the following directive in /etc/ssh/sshd_config: 
    Banner /etc/issue.net
    Once configured, this banner will be displayed to all users connecting via SSH before login.
  5. Post-Login Banner
    A post-login banner is displayed after a user successfully authenticates to the system. This message appears once the user has logged in and before they begin interacting with the command-line shell.
  6. Purpose of Post-Login Banners
    Post-login banners are commonly used to display operational information and system status details. Instead of legal warnings, these banners typically provide helpful system insights to administrators and users.
  7. Common Post-Login Banner Configuration
    Post-login messages are typically managed using:
    • /etc/motd (Message of the Day)
    • /etc/update-motd.d/ scripts (dynamic messages)
    These files allow administrators to display static messages or dynamically generated system information when users log in.
  8. Dynamic Post-Login Information
    Administrators often configure dynamic MOTD scripts to display real-time system data such as:
    • System uptime
    • CPU load
    • Memory usage
    • Disk space availability
    • Number of active users
    • System hostname and network information
    This helps administrators quickly understand the system state immediately after login.
  9. Choosing the Right Banner Type
    Both banner types serve different roles in system administration:
    • Pre-login banners are best used for security warnings and legal notices.
    • Post-login banners are ideal for operational information and system status.
    In many environments, administrators configure both types to provide security warnings before login and useful system information after authentication.
Completion check: You should be able to explain the difference between pre-login and post-login SSH banners, when each type is used, and which system files are responsible for displaying them.

  2. Install required tools sudo apt install -y figlet toilet neofetch ruby
  3. Install lolcat for colored output sudo gem install lolcat
  4. Create dashboard script sudo nano /etc/update-motd.d/99-dashboard
  5. Add the following script 
    #!/usr/bin/env bash

HOST=$(hostname)
IP=$(hostname -I | awk '{print $1}')
CPU=$(nproc)
MEM_TOTAL=$(free -h | awk '/Mem:/ {print $2}')
MEM_USED=$(free -h | awk '/Mem:/ {print $3}')
DISK=$(df -h / | awk 'NR==2 {print $5}')
UPTIME=$(uptime -p | sed 's/up //')
TIME=$(date)

if command -v docker >/dev/null 2>&1; then
    DOCKER_COUNT=$(docker ps -q | wc -l)
else
    DOCKER_COUNT="Docker not installed"
fi

clear

figlet -f slant "DEVOPS SERVER" | lolcat

echo "" | lolcat

printf "══════════════════════════════════════════════════════════════\n" | lolcat
printf "                 🚀 SERVER INFORMATION\n" | lolcat
printf "══════════════════════════════════════════════════════════════\n" | lolcat

printf " Hostname        : %s\n" "$HOST" | lolcat
printf " IP Address      : %s\n" "$IP" | lolcat
printf " Login Time      : %s\n" "$TIME" | lolcat

echo "" | lolcat

printf "══════════════════════════════════════════════════════════════\n" | lolcat
printf "                 💻 SYSTEM RESOURCES\n" | lolcat
printf "══════════════════════════════════════════════════════════════\n" | lolcat

printf " CPU Cores       : %s\n" "$CPU" | lolcat
printf " Memory Usage    : %s / %s\n" "$MEM_USED" "$MEM_TOTAL" | lolcat
printf " Disk Usage      : %s\n" "$DISK" | lolcat
printf " Uptime          : %s\n" "$UPTIME" | lolcat

echo "" | lolcat

printf "══════════════════════════════════════════════════════════════\n" | lolcat
printf "                 🐳 PLATFORM SERVICES\n" | lolcat
printf "══════════════════════════════════════════════════════════════\n" | lolcat

printf " Docker Containers Running : %s\n" "$DOCKER_COUNT" | lolcat

echo "" | lolcat

printf "══════════════════════════════════════════════════════════════\n" | lolcat
printf "⚠ Authorized access only. All activities are monitored.\n" | lolcat
printf "══════════════════════════════════════════════════════════════\n" | lolcat
