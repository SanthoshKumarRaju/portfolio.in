// Kubernetes

Pod Deployments

• localhost

Hands-on with K8s primitives — rolling updates, rollbacks, HPA and liveness/readiness probes.

Setup Steps
  1. Create Deployment with 3 replicas and probes
  2. HPA: kubectl autoscale deployment myapp --min=2 --max=10 --cpu-percent=50
  3. Rolling update: kubectl set image deployment/myapp myapp=myapp:v2
  4. Monitor: kubectl rollout status deployment/myapp
  5. Rollback: kubectl rollout undo deployment/myapp
hpa.yaml yaml
apiVersion: autoscaling/v2 kind: HorizontalPodAutoscaler metadata: {name: myapp-hpa} spec: scaleTargetRef: apiVersion: apps/v1 kind: Deployment name: myapp minReplicas: 2 maxReplicas: 10 metrics: - type: Resource resource: name: cpu target: {type: Utilization, averageUtilization: 50}
