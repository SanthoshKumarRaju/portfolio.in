Linux

Basic Linux Commands

• local lab
Summary

This project is designed to help you get hands-on experience with essential Linux commands and concepts. You'll set up a local Ubuntu VM using VMware or VirtualBox, then practice navigating the filesystem, managing files and permissions, checking system resources, and troubleshooting services using logs. By completing the structured tasks, you'll build a solid foundation in Linux that will be valuable for any admin or DevOps role.

Task Checklist

Goal: Install the virtualization tool required to run Ubuntu.

  1. Open your browser and go to VMware official downloads page for Workstation Player .
  2. Select the installer matching your host OS and download it to Downloads Now.
  3. Run the installer with administrator permissions and accept the license agreement.
  4. Keep default install path and default feature options unless your lab has custom requirements.
  5. Complete installation, restart your host machine if prompted, then launch VMware once.
  6. Confirm the app opens to the VMware home dashboard without any startup error.
Completion check: VMware opens and shows the home screen.

Goal: Get the OS image needed for VM installation.

  1. Go to the Ubuntu official download page and choose Ubuntu 22.04 LTS Desktop .
  2. Select the OS version you prefer; The recommended version - 22.04 LTS Desktop - click on Download.
  3. Start the ISO download and wait for the file to fully complete before closing browser.
  4. Move the ISO to a stable folder path such as D:\ISO\ubuntu-22.04.iso.
  5. Open file properties and confirm the file size is not zero and extension is .iso.
  6. Recommended: compare SHA256 checksum with Ubuntu website to verify file integrity.
Completion check: ISO file is fully downloaded and accessible.

Goal: create a VM with practical resources for Linux learning.

  1. Click Create a New Virtual Machine from VMware home screen.
  2. Choose Installer disc image file (iso) and browse to your Ubuntu ISO path.
  3. Set guest OS type to Linux and version to Ubuntu 64-bit.
  4. Enter VM name such as Ubuntu-Lab and select a folder with enough free space.
  5. Allocate disk size 40 GB (single file is fine for a local learning VM).
  6. Open hardware settings and configure RAM at least 4 GB, CPU 2 cores, network NAT.
  7. Save configuration and power on the VM to confirm it boots to Ubuntu installer.
Completion check: VM boots from ISO and Ubuntu installer screen appears.

Goal: complete OS installation and first boot.

  1. Select Install Ubuntu in the booted installer screen.
  2. Pick your keyboard layout and test typing in the installer input box.
  3. Choose Normal installation and enable downloading updates during setup.
  4. Select Erase disk and install Ubuntu only because this is an isolated VM disk.
  5. Set timezone correctly, create username, hostname, and a strong password.
  6. Allow installation to finish, then click restart and remove ISO if prompted.
  7. Log in after reboot and confirm desktop, terminal, and settings open normally.
Completion check: You can log in to Ubuntu desktop normally.

Goal: keep OS updated and create a safe lab directory.

  1. Open terminal using Ctrl + Alt + T immediately after first login.
  2. Refresh package index: sudo apt update and check for successful completion.
  3. Apply upgrades: sudo apt upgrade -y and wait until prompt returns.
  4. Install key tools: sudo apt install -y curl wget git htop tree.
  5. Create your practice folder and enter it: mkdir -p /tmp/lab && cd /tmp/lab.
  6. Confirm location with pwd and verify output shows /tmp/lab.
Completion check: package update completed and /tmp/lab exists.

  • find
    Searches for files and directories in the filesystem.
    Syntax
    find <path> [options] <expression>
    Common Options
    -name → search by file name
    -type f → search files only
    -type d → search directories only
    -size → search by file size
    Examples
    find . -name file.txt
    find /home -type f -name "*.log"
    Output Example
    ./logs/error.log
      ./logs/system.log
  • grep
    Searches for text patterns inside files.
    Syntax
    grep [options] "pattern" <file>
    Common Options
    -i → ignore case
    -r → search recursively
    -n → show line numbers
    -v → show lines that do NOT match
    Examples
    grep "error" log.txt
    grep -i "warning" log.txt
    Output Example
    45:error connecting to database
      78:error timeout
  • find + grep
    Combines file searching and text searching.
    Example
    find . -name "*.log" | grep error
    Description
    Finds all log files and filters results containing "error".
Practical Examples
find /var/log -name "*.log"
grep -i "failed" auth.log
grep -rn "password" /etc
Output Example
/etc/ssh/sshd_config:34:PasswordAuthentication yes
      /etc/pam.d/common-password:12:password required

    Syntax
    chmod 755 <file>
    Example

    7 → read + write + execute (owner)
    5 → read + execute (group)
    5 → read + execute (others)

    Syntax
    chmod +x <file>
    Example
    chmod +x script.sh
    Description
    Makes the file executable so it can run as a script.
  • chown

    Syntax
    chown <user> <file>
    Example
    chown user file.txt
    Description
    Transfers file ownership to another user.
  • chown user:group
    Changes both owner and group of a file.
    Syntax
    chown <user>:<group> <file>
    Example
    chown user:devteam file.txt
    Description
    Assigns both user ownership and group ownership.

    Displays detailed file information including permissions, owner, and group.
    Syntax
    ls -l
    Example
    ls -l script.sh
    Output Example
    -rwxr-xr-x 1 santhosh developers 512 Aug 10 12:00 script.sh

  • >
    Redirects command output to a file (overwrite mode).
    Syntax
    command > file.txt
    Example
    ls > files.txt
    Description
    Creates a file and writes command output to it. If the file exists, it will be overwritten.
    Output Example
    files.txt
      --------
      file1.txt
      file2.txt
      script.sh
  • >>
    Redirects command output to a file in append mode.
    Syntax
    command >> file.txt
    Example
    date >> log.txt
    Description
    Adds output to the end of the file without deleting existing content.
  • |
    Sends the output of one command as input to another command.
    Syntax
    command1 | command2
    Example
    ls | grep ".txt"
    Description
    Filters the output of the first command using another command.
    Output Example
    notes.txt
      data.txt
Example Workflow
ls > files.txt
echo "backup completed" >> log.txt
cat files.txt | grep ".txt"
Output Example
file1.txt
      file2.txt

  • useradd
    Creates a new user account on the system.
    Syntax
    sudo useradd username
    Example
    sudo useradd devuser
    Common Options
    -m → create home directory
    -s → specify login shell
  • passwd
    Sets or changes the password for a user.
    Syntax
    sudo passwd username
    Example
    sudo passwd devuser
    Description
    Prompts to enter a new password for the user.
  • usermod
    Modifies an existing user account.
    Syntax
    sudo usermod -aG group username
    Example
    sudo usermod -aG sudo devuser
    Option Breakdown
    -a → append user to group
    -G → specify group
    Description
    Adds the user to the sudo group for administrative privileges.
  • userdel
    Deletes a user account from the system.
    Syntax
    sudo userdel username
    Example
    sudo userdel devuser
    Common Option
    -r → remove user home directory
Example Workflow
sudo useradd -m devuser
sudo passwd devuser
sudo usermod -aG sudo devuser

  • ssh
    Securely connects to a remote server using SSH.
    Syntax
    ssh user@server
    Example
    ssh ubuntu@192.168.1.10
    Description
    Opens a secure remote shell session.
    Useful Option
    -i → specify SSH private key
  • scp
    Securely copies files between local and remote systems.
    Syntax
    scp file user@server:/path
    Example
    scp backup.tar.gz user@192.168.1.10:/backup
    Common Options
    -r → copy directories recursively
    -i → specify SSH key
  • rsync
    Synchronizes files and directories between systems efficiently.
    Syntax
    rsync -av folder server:/backup
    Option Breakdown
    -a → archive mode (preserve permissions)
    -v → verbose output
    Example
    rsync -av project/ user@server:/backup/project
    Description
    Transfers only changed files, making backups faster.
Example Workflow

scp config.yml user@server:/etc/app
rsync -av project/ user@server:/backup/project
