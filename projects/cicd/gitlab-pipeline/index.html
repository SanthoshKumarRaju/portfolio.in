GitLab CI Pipeline

• server

End-to-end GitLab CI pipeline: lint → test → docker build → push → deploy via SSH. Manual gate for production with environment tracking.

Setup Steps
  1. Create .gitlab-ci.yml in repo root
  2. Set CI/CD variables: REGISTRY_USER, REGISTRY_PASS, DEPLOY_HOST, DEPLOY_KEY
  3. Push to main branch to trigger pipeline
  4. Review stages in GitLab → CI/CD → Pipelines
  5. Approve manual production deploy gate
.gitlab-ci.yml yaml
stages: [test, build, deploy] variables: IMAGE: "${CI_REGISTRY_IMAGE}:${CI_COMMIT_SHORT_SHA}" test: stage: test script: [npm ci, npm test] build: stage: build script: - docker build -t $IMAGE . - docker push $IMAGE deploy-prod: stage: deploy when: manual script: - ssh deploy@$DEPLOY_HOST "docker service update --image $IMAGE app_web"
